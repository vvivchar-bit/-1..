# Система пропускного контролю в 18 корпус

Навчальний вебпрототип до **Лабораторної роботи №1** з теми **«Виявлення та характеристика вимог до програмного продукту»**.

## Варіант

**№7 — Система пропускного контролю в 18 корпус.**

## Короткий опис

Проєкт демонструє базову логіку пропускного контролю в навчальному корпусі. Система дозволяє перевіряти право доступу особи до 18 корпусу, вести список користувачів, блокувати або активувати пропуски та автоматично формувати журнал входів і виходів.

Прототип працює без серверної частини: усі дані зберігаються в **LocalStorage** браузера.

## Основні можливості

- перевірка доступу за номером пропуску або ПІБ;
- додавання студентів, викладачів, працівників і відвідувачів;
- блокування та активація пропусків;
- перевірка терміну дії пропуску;
- фіксація входу та виходу;
- автоматичне ведення журналу подій;
- експорт журналу у форматі CSV;
- відновлення демонстраційних даних.

## Як запустити

1. Завантажити або клонувати репозиторій.
2. Відкрити файл `index.html` у будь-якому сучасному браузері.
3. Для тестування використати демо-пропуски:
   - `ST-001` — активний студентський пропуск;
   - `TC-014` — активний пропуск викладача;
   - `WK-022` — активний пропуск працівника;
   - `VS-101` — активний пропуск відвідувача;
   - `ST-009` — заблокований студентський пропуск.

## Технології

- HTML5;
- CSS3;
- JavaScript;
- LocalStorage.

## Звіт

Звіт до лабораторної роботи розміщено в папці `docs/` у форматах **PDF** та **DOCX**.

## Призначення роботи

Цей проєкт є демонстраційним і створений для підтвердження вимог, описаних у звіті лабораторної роботи. Він не є промисловою системою безпеки, але показує основні функції майбутньої системи пропускного контролю.
